## 1. 数据字段（Property）

### 1.1 固定信息字段
这些字段通常在对象创建时设置,后续很少变化。

#### 工单编号
- **类型**: 文本（String）
- **说明**: 工单的唯一标识,系统自动生成
- **示例**: WO-2024-0318-001
- **约束**: 唯一,不可重复

#### VIN码
- **类型**: 文本（String）
- **说明**: 车辆识别代号,全球唯一的17位编码
- **示例**: LSGC53E07DS123456
- **约束**: 必填,长度必须为17位

#### 品牌
- **类型**: 文本（String）
- **说明**: 车辆品牌
- **示例**: 理想、蔚来、特斯拉

#### 车型
- **类型**: 文本（String）
- **说明**: 车辆具体型号
- **示例**: L9 MAX、ES8、Model 3

#### 车牌号
- **类型**: 文本（String）
- **说明**: 车辆牌照号码
- **示例**: 沪A12345

#### 车主姓名
- **类型**: 文本（String）
- **说明**: 车辆所有人姓名
- **示例**: 张三

#### 联系电话
- **类型**: 文本（String）
- **说明**: 车主联系方式
- **示例**: 13812345678
- **约束**: 必填,必须是有效手机号

#### 故障现象
- **类型**: 文本（String）
- **说明**: 客户报修时描述的故障情况
- **示例**: "发动机异响,加速无力,仪表盘显示故障灯"

#### 工单类型
- **类型**: 枚举（Enum）
- **说明**: 维修服务的类别
- **可选值**: 保养 | 故障 | 事故 | 召回
- **示例**: 故障

#### 创建时间
- **类型**: 时间戳（Timestamp）
- **说明**: 工单创建的时间,系统自动记录
- **示例**: 2024-03-18 14:30:00

#### 创建人
- **类型**: 文本（String）
- **说明**: 创建工单的调度员或客服姓名
- **示例**: 李调度

## 2. 计算规则（Function）

计算规则会根据数据字段自动计算结果,不需要人工输入。

### 2.1. 工单紧急程度

**业务目的**: 根据工单类型和等待时长自动判断优先级,确保紧急工单优先处理

**计算逻辑**:
1. 如果工单已派单,返回"普通"
2. 如果工单未派单:
   - 事故工单: 等待>30分钟 → 紧急; 否则 → 高优先级
   - 故障工单: 等待>60分钟 → 紧急; 否则 → 高优先级
   - 保养工单: 等待>120分钟 → 高优先级; 否则 → 普通
   - 召回工单: → 普通

**输入数据**:
- 工单类型
- 创建时间
- 派单时间
- 当前时间

**输出结果**: 紧急 | 高优先级 | 普通

**使用场景**:
- 工单列表中高亮显示紧急工单
- 触发超时告警自动化
- 调度系统优先级排序

**示例**:
```
工单A:
  - 类型: 事故
  - 创建时间: 14:00
  - 当前时间: 14:40
  - 等待时长: 40分钟
  - 结果: 紧急 (超过30分钟)

工单B:
  - 类型: 保养
  - 创建时间: 10:00
  - 当前时间: 11:00
  - 等待时长: 60分钟
  - 结果: 普通 (未超过120分钟)
```

---

### 2.2. 维修复杂度

**业务目的**: 根据故障现象关键词自动判断维修难度,用于技师匹配和工时费用计算

**计算逻辑**:
- 包含复杂关键词 → 复杂
  - 关键词: 发动机、变速箱、电池组、电机、刹车系统、转向系统、悬挂系统
- 包含一般关键词 → 一般
  - 关键词: 空调、音响、车窗、座椅、仪表盘、灯光、胎压
- 包含简单关键词 → 简单
  - 关键词: 轮胎、雨刷、保养、玻璃水、清洁
- 无匹配关键词 → 一般(默认)

**输入数据**:
- 故障现象描述

**输出结果**: 复杂 | 一般 | 简单

**使用场景**:
- 技师匹配(根据技师技能等级)
- 工时费用计算(复杂150元/小时,一般100元/小时,简单80元/小时)
- 预估维修时长

**示例**:
```
故障描述: "发动机异响,加速无力"
→ 包含"发动机" → 复杂

故障描述: "空调不制冷"
→ 包含"空调" → 一般

故障描述: "更换雨刷器"
→ 包含"雨刷" → 简单
```

---

### 2.3. 配件库存状态

**业务目的**: 实时检查推荐配件的库存情况,提前预警缺货

**计算逻辑**:
1. 如果无需配件 → 库存充足
2. 如果关键配件缺货 或 所有配件都缺货 → 缺货
3. 如果部分配件缺货 → 库存不足
4. 否则 → 库存充足

**输入数据**:
- 推荐配件清单(逗号分隔)
- 配件系统实时库存(通过API获取)

**输出结果**: 库存充足 | 库存不足 | 缺货

**使用场景**:
- 工单创建时提前告警缺货
- 触发配件补货流程
- 派单时考虑配件可用性

**示例**:
```
推荐配件: "空气滤芯, 机油, 机滤"
库存情况:
  - 空气滤芯: 5个 ✓
  - 机油: 10桶 ✓
  - 机滤: 0个 ✗
→ 库存不足

推荐配件: "发动机总成"
库存情况:
  - 发动机总成: 0个 ✗ (关键配件)
→ 缺货
```

### 2.4. 是否完工超时

**业务目的**: 对比实际工时与维修手册标准工时,判断是否超时

**计算逻辑**:
- 实际工时 ≤ 标准工时 × 150% → 按时完工
- 标准工时 × 150% < 实际工时 ≤ 标准工时 × 200% → 轻微超时
- 实际工时 > 标准工时 × 200% → 严重超时

**输入数据**:
- 实际工时
- 车型
- 故障现象(用于查询维修手册标准工时)

**输出结果**: 按时完工 | 轻微超时 | 严重超时

**使用场景**:
- 技师绩效考核
- 触发超时告警
- 质量分析

**示例**:
```
车型: L9 MAX
故障: 更换空调压缩机
标准工时: 2.0小时

实际工时: 2.5小时
→ 2.5 ≤ 2.0 × 1.5 (3.0) → 按时完工

实际工时: 3.5小时
→ 3.0 < 3.5 ≤ 4.0 → 轻微超时

实际工时: 5.0小时
→ 5.0 > 4.0 → 严重超时
```

### 2.5. 是否一次修好(返修判断)

**业务目的**: 自动检测是否为返修工单,触发质量分析和特殊处理

**计算逻辑**:
1. 查询同一VIN码在30天内的历史工单
2. 提取当前故障和历史故障的关键词
3. 如果关键词有重合 → 返修
4. 否则 → 正常

**输入数据**:
- VIN码
- 故障现象
- 当前工单创建时间

**输出结果**: 正常 | 返修

**使用场景**:
- 触发返修告警
- 质量分析
- 技师绩效考核

**示例**:
```
当前工单:
  - VIN: LSGC53E07DS123456
  - 故障: "发动机异响"
  - 创建时间: 2024-03-18

历史工单(30天内):
  - 2024-03-01: "发动机抖动,有异响" ✗ (关键词重合: "发动机")
→ 返修

当前工单:
  - VIN: LSGC53E07DS123456
  - 故障: "空调不制冷"
  - 创建时间: 2024-03-18

历史工单(30天内):
  - 2024-03-01: "发动机异响" ✓ (无关键词重合)
→ 正常
```

### 2.6. 车主满意度预测

**业务目的**: 根据服务质量指标预测客户满意度,低满意度主动回访

**计算逻辑**:
评分权重: 服务态度(40%) > 一次修好(30%) > 费用(20%) > 耗时(10%)

1. 服务态度得分 = 服务态度评分 × 20 (1-5星 → 20-100分)
2. 一次修好得分 = 正常100分, 返修0分
3. 费用得分 = <1000元:100分, 1000-3000元:80分, >3000元:60分
4. 耗时得分 = <4小时:100分, 4-8小时:80分, >8小时:60分
5. 总分 = 各项得分 × 权重
6. 满意度分级:
   - ≥85分 → 很满意
   - ≥70分 → 满意
   - ≥60分 → 一般
   - <60分 → 不满意

**输入数据**:
- 服务态度评分(1-5星)
- 是否返修(正常/返修)
- 维修费用(元)
- 总耗时(小时)

**输出结果**: 很满意 | 满意 | 一般 | 不满意

**使用场景**:
- 低满意度预警
- 主动回访
- 服务质量分析

**示例**:
```
技师A的工单:
  - 服务态度: 5星 (100分)
  - 一次修好: 正常 (100分)
  - 费用: 800元 (100分)
  - 耗时: 3小时 (100分)
  - 总分: 100 × 0.4 + 100 × 0.3 + 100 × 0.2 + 100 × 0.1 = 100
  → 很满意

技师B的工单:
  - 服务态度: 3星 (60分)
  - 一次修好: 返修 (0分)
  - 费用: 3500元 (60分)
  - 耗时: 9小时 (60分)
  - 总分: 60 × 0.4 + 0 × 0.3 + 60 × 0.2 + 60 × 0.1 = 42
  → 不满意 (触发低满意度告警)
```

### 2.7. 维修成本明细

**业务目的**: 自动计算配件成本、工时费用、总费用,生成电子账单

**计算逻辑**:
1. 配件成本 = Σ(每个配件标准单价 × 数量)
2. 工时费用 = 实际工时 × 工时单价
   - 工时单价根据维修复杂度:
     - 复杂: 150元/小时
     - 一般: 100元/小时
     - 简单: 80元/小时
3. 总费用 = 配件成本 + 工时费用

**输入数据**:
- 实际使用的配件清单
- 实际工时
- 维修复杂度

**输出结果**:
- 配件成本(元)
- 工时费用(元)
- 总费用(元)

**使用场景**:
- 完工时自动计算费用
- 生成电子账单
- 成本分析

**示例**:
```
配件清单: "空气滤芯x1, 机油5W-30x4L, 机滤x1"
配件单价:
  - 空气滤芯: 80元
  - 机油5W-30: 50元/L × 4L = 200元
  - 机滤: 30元
配件成本: 80 + 200 + 30 = 310元

实际工时: 2.5小时
维修复杂度: 一般 (100元/小时)
工时费用: 2.5 × 100 = 250元

总费用: 310 + 250 = 560元
```

## 3. 操作按钮（Action）

操作按钮是用户与系统交互的入口,每个按钮对应一个业务操作。

### 3.1. 创建维修工单

**操作类型**: 创建

**业务场景**: 客户报修时,调度员或客服在系统中创建新的维修工单

**需要填写的信息**:
- **VIN码** (文本, 必填): 车辆识别代号,17位
- **车型** (文本, 必填): 如 L9 MAX
- **车主姓名** (文本, 必填): 客户姓名
- **联系电话** (文本, 必填): 客户手机号
- **故障现象** (文本, 必填): 客户描述的问题
- **工单类型** (枚举, 必填): 保养/故障/事故/召回
- **品牌** (文本, 选填): 如 理想
- **车牌号** (文本, 选填): 如 沪A12345
- **行驶里程** (整数, 选填): 如 35000公里

**执行后会发生什么**:
1. 系统自动生成工单编号(如 WO-2024-0318-001)
2. 创建时间 = 当前时间
3. 创建人 = 当前登录用户
4. 工单状态 = "待派单"
5. 触发Automation "工单创建后自动推荐维修方案"

**谁可以执行**:
- 调度员: 可以创建所有类型工单
- 客服: 可以创建所有类型工单

---

### 3.2. 分配技师(派单)

**操作类型**: 更新

**业务场景**: 调度员根据技师技能、位置、负载情况,将工单派给合适的技师

**需要填写的信息**:
- **分配的技师** (文本, 必填): 从技师列表中选择
- **派单备注** (文本, 选填): 如"需要高级技师","紧急工单"

**前置条件**:
- 工单状态必须是"待派单"

**执行后会发生什么**:
1. 工单的"分配的技师"字段更新
2. 派单时间 = 当前时间
3. 工单状态 → "进行中"
4. 发送通知到技师APP: "您有新的维修工单,请及时处理"

**谁可以执行**:
- 调度员: 可以派单

---

### 3.3. 技师接单

**操作类型**: 更新

**业务场景**: 技师在APP中查看工单详情,确认接单

**需要填写的信息**:
- **预计到达时间** (时间, 必填): 技师预计车辆进店的时间
- **接单备注** (文本, 选填): 如"30分钟后到达"

**前置条件**:
- 工单已派单给该技师
- 工单状态是"进行中"

**执行后会发生什么**:
1. 技师接单时间 = 当前时间
2. 发送通知到调度员: "技师XX已接单,预计XX时间到达"

**谁可以执行**:
- 分配给该工单的技师

---

### 3.4. 车辆进店

**操作类型**: 更新

**业务场景**: 客户车辆到达维修门店,技师在APP中签到

**需要填写的信息**:
- **进店位置** (文本, 选填): 如"1号工位"
- **进店备注** (文本, 选填): 如"客户同时在等候区"

**前置条件**:
- 技师已接单

**执行后会发生什么**:
1. 车辆进店时间 = 当前时间
2. 发送通知到车主: "您的车辆已进店,技师正在准备维修"

**谁可以执行**:
- 分配给该工单的技师

### 3.5. 开始维修

**操作类型**: 更新

**业务场景**: 技师完成故障诊断,开始实际维修作业

**需要填写的信息**:
- **实际故障诊断** (文本, 必填): 技师现场确认的故障(可能与客户描述不同)
- **维修照片** (文本, 选填): 故障照片URL

**前置条件**:
- 车辆已进店

**执行后会发生什么**:
1. 开始维修时间 = 当前时间
2. 如果实际故障与推荐方案不符,可重新获取推荐

**谁可以执行**:
- 分配给该工单的技师

### 3.6. 提交完工

**操作类型**: 更新

**业务场景**: 技师完成维修作业,提交完工信息

**需要填写的信息**:
- **实际使用的配件清单** (文本, 必填): 如"空气滤芯x1, 机油x4L"
- **实际工时** (小数, 必填): 如 2.5小时
- **维修照片** (文本, 必填): 完工照片URL
- **完工说明** (文本, 选填): 如"已完成保养,建议下次3个月后再保养"

**前置条件**:
- 已开始维修

**执行后会发生什么**:
1. 完工时间 = 当前时间
2. 调用Function "维修成本明细" 自动计算费用
3. 工单状态 → "已完工"
4. 触发Automation "完工后自动计算费用并通知客户"
   - 生成电子账单(PDF)
   - 发送短信通知车主: "您的车辆已维修完成,费用XX元,请尽快取车"

**谁可以执行**:
- 分配给该工单的技师

### 3.7. 审核关闭

**操作类型**: 更新

**业务场景**: 质检人员或主管审核完工质量,关闭工单

**需要填写的信息**:
- **服务态度评分** (1-5星, 必填): 对技师服务态度的评价
- **审核意见** (文本, 选填): 如"质量合格","需要改进XX"

**前置条件**:
- 工单状态是"已完工"

**执行后会发生什么**:
1. 关闭时间 = 当前时间
2. 审核人 = 当前登录用户
3. 工单状态 → "已关闭"
4. 触发Automation "审核关闭后自动触发满意度调查"
   - 调用Function "车主满意度预测"
   - 如果预测满意度 < "满意" → 告警客服主管主动回访
   - 发送满意度调查短信给车主
   - 更新技师工作量统计

**谁可以执行**:
- 审核人(质检、主管)

### 3.9. 技师拒单/转派

**操作类型**: 业务操作

**业务场景**: 技师接单后发现无法处理(技能不匹配、时间冲突、距离太远等),申请转派

- **拒单原因** (枚举, 必填): 技能不匹配/时间冲突/距离太远/其他

**前置条件**:

**执行后会发生什么**:
1. 工单状态 → "待派单"
2. 清空"分配的技师"字段
3. 触发Automation "技师拒单后自动重新派单提醒"
   - 发送通知到调度员: "技师XX拒单,工单YY需要重新分配"
   - 在调度系统中标记为"优先派单"
   - 记录拒单历史(用于技师绩效分析)

**谁可以执行**:
- 分配给该工单的技师

### 3.10. 取消工单

**操作类型**: 业务操作

**业务场景**: 客户取消预约、重复工单、误创建等情况

- **取消原因** (枚举, 必填): 客户取消/重复工单/误创建/其他

**前置条件**:
- 工单状态是"待派单"或"进行中"
- 车辆未进店(未开始维修)

**执行后会发生什么**:
1. 工单状态 → "已关闭(已取消)"
2. 如果已派单,发送通知到技师: "工单XX已取消,无需处理"
3. 记录取消日志

**谁可以执行**:
- 调度员: 可以取消任何工单
- 客服: 可以取消任何工单

### 3.11. 申请技术支持

**业务场景**: 技师遇到疑难故障,需要专家协助

- **支援原因** (文本, 必填): 描述遇到的技术问题

**前置条件**:
- 工单状态是"进行中"

**执行后会发生什么**:

   - 发送支援请求到对应专家: "技师XX需要协助,工单YY,故障:ZZ"

   - 如果30分钟内专家未响应 → 自动升级通知技术主管

**谁可以执行**:
- 分配给该工单的技师

4. 记录返修日志(用于质量分析)

**谁可以执行**:
- 技师: 可以登记返修

## 4. 自动化规则（Automation）

自动化规则在后台运行,当满足特定条件时自动执行操作。

### 4.1. 工单超时自动提醒

**自动化类型**: 异常响应

**业务价值**: 确保紧急工单及时派单,避免客户长时间等待

**触发条件**:
Function "工单紧急程度" 返回 "紧急"
(事故工单创建30分钟未派单,或故障工单60分钟未派单)

**自动执行流程**:
1. 发送告警通知
   - 企业微信 → 调度主管
   - 消息: "【超时告警】工单 XX 已超时未派单,请尽快处理"
2. 在调度系统中高亮显示该工单
3. 记录超时日志

**安全提示**: 限流配置 - 同一工单30分钟内最多提醒1次

---

### 4.2. 维修严重超时自动上报

**自动化类型**: 异常响应

**业务价值**: 及时发现维修超时问题,避免影响客户满意度

**触发条件**:
Function "是否完工超时" 返回 "严重超时"
(实际工时 > 标准工时 × 200%)

**自动执行流程**:
1. 发送通知
   - 企业微信 → 技师主管
   - 消息: "【超时告警】技师XX的工单YY严重超时,已用时ZZ小时(标准WW小时)"
2. 工单标记为"异常"
3. 自动触发质检流程

---

### 4.3. 配件缺货自动通知

**自动化类型**: 异常响应

**业务价值**: 提前预警配件缺货,避免影响维修进度

**触发条件**:
Function "配件库存状态" 返回 "缺货"

**自动执行流程**:
1. 发送缺货通知
   - 企业微信 → 配件管理员
   - 消息: "【缺货】工单XX需要配件YY,当前库存为0"
2. 在工单中标记"等待配件"
3. 创建配件补货提醒

---

### 4.4. 疑似返修自动提醒

**自动化类型**: 异常响应

**业务价值**: 及时发现返修问题,触发质量分析和改进

**触发条件**:
Function "是否一次修好" 返回 "返修"
(同一车辆30天内相同故障)

**自动执行流程**:
1. 发送返修告警
   - 企业微信 → 质检主管、原技师
   - 消息: "【返修告警】车辆XX疑似返修,原工单YY,请关注"
2. 工单自动关联原工单
3. 触发质量分析流程

---

### 4.5. 工单创建后自动推荐维修方案

**自动化类型**: 流程编排

**业务价值**: 利用AI和历史数据自动推荐维修方案,提升技师效率

**触发条件**:
Action "创建维修工单" 被执行

**自动执行流程**:
1. 调用Function "AI推荐维修方案"(实际需要外部AI服务)
   - 根据车型、故障现象、历史数据生成推荐方案
2. 将推荐结果写入工单字段
   - 推荐的维修方案
   - 预估工时
   - 推荐配件清单
3. 调用Function "配件库存状态"
   - 检查推荐配件的库存情况
4. 如果有缺货,自动通知配件管理员

---

### 4.6. 技师拒单后自动重新派单提醒

**自动化类型**: 流程编排

**业务价值**: 确保拒单工单及时重新派单,避免流程中断

**触发条件**:
Action "技师拒单/转派" 被执行

**自动执行流程**:
1. 工单状态恢复为"待派单"
2. 发送通知
   - 企业微信 → 调度员
   - 消息: "【需重新派单】技师XX拒单,工单YY需要重新分配,拒单原因:ZZ"
3. 在调度系统中标记为"优先派单"
4. 记录拒单历史(用于技师绩效分析)

---

### 4.7. 完工后自动计算费用并通知客户

**自动化类型**: 流程编排

**业务价值**: 自动化费用计算和客户通知,减少人工操作

**触发条件**:
Action "提交完工" 被执行

**自动执行流程**:
1. 调用Function "维修成本明细"
   - 自动计算配件成本、工时费用、总费用
2. 生成电子账单(PDF)
3. 发送完工通知
   - 短信 → 车主手机
   - 消息: "您的车辆XX已维修完成,费用YY元,请尽快取车"
4. 更新工单状态为"待审核"

---

### 4.8. 审核关闭后自动触发满意度调查

**自动化类型**: 流程编排

**业务价值**: 主动收集客户反馈,低满意度及时回访

**触发条件**:
Action "审核关闭" 被执行

**自动执行流程**:
1. 调用Function "车主满意度预测"
   - 预测客户满意度
2. 如果预测满意度 < "满意"
   - 发送告警 → 客服主管
   - 消息: "工单XX客户满意度预测较低,建议主动回访"
3. 发送满意度调查
   - 短信 → 车主(包含调查问卷链接)
4. 记录工单完成
   - 更新技师工作量统计
   - 更新车辆维修历史

---

### 4.9. 申请技术支持后自动派遣专家

**业务价值**: 快速响应技术支援请求,确保疑难故障得到解决

Action "申请技术支持" 被执行

**自动执行流程**:

3. 在工单中记录支援状态
4. 如果30分钟内专家未响应
   - 自动升级通知技术主管

### 4.10. 每日工单统计报告

## 5. 业务流程示意图

```
正常维修流程:

客户报修
   ↓
[创建维修工单] (调度员/客服)
   ↓
Automation: 自动推荐维修方案 (AI)
   ↓
Function: 检查配件库存状态
   ↓
[分配技师] (调度员)
   ↓
Function: 判断工单紧急程度 → 如果超时 → Automation: 超时告警
   ↓
[技师接单] (技师)
   ↓
[车辆进店] (技师)
   ↓
[开始维修] (技师)
   ↓
Function: 判断维修复杂度 → 影响工时费用
   ↓
维修过程中如遇困难 → [申请技术支持] → Automation: 自动派遣专家
   ↓
[提交完工] (技师)
   ↓
Automation: 自动计算费用并通知客户
   ↓
Function: 判断是否完工超时 → 如果严重超时 → Automation: 超时上报
   ↓
[审核关闭] (审核人)
   ↓
Function: 判断是否返修 → 如果返修 → Automation: 返修告警
Function: 预测车主满意度 → 如果低满意度 → Automation: 主动回访
Automation: 发送满意度调查

定时任务每天/每周/每月自动执行统计报告
```

## 6. 典型使用场景

### 场景1: 紧急事故工单快速响应

**情况**: 客户车辆发生事故无法行驶,下午2点报修

**系统响应**:
1. 14:00 - 客服创建工单,类型="事故"
2. 14:01 - AI自动推荐维修方案,检查配件库存
3. 14:05 - 调度员派单给技师王工
4. 14:10 - 技师王工接单,预计14:40到达
5. 14:30 - 系统检测到工单未完成派单超过30分钟
   - Function "工单紧急程度" 返回"紧急"
   - Automation触发: 发送告警到调度主管(但此时已派单,不再告警)
6. 14:40 - 车辆拖到门店,技师签到"车辆进店"
7. 14:45 - 技师开始维修
8. 17:00 - 技师提交完工,实际工时2.25小时
   - Function自动计算费用: 配件1200元 + 工时337.5元(2.25×150元/小时,复杂维修) = 1537.5元
   - Automation: 发送短信给客户"您的车辆已维修完成,费用1537.5元"
9. 17:30 - 审核人关闭工单,服务态度评分5星
   - Function预测满意度: 85分 → "很满意"
   - Automation: 发送满意度调查给客户

**最终结果**: 从报修到完工3小时,客户满意度高,流程顺畅

---

### 场景2: 配件缺货智能预警

**情况**: 客户报修空调压缩机故障,但配件缺货

**系统响应**:
1. 10:00 - 客服创建工单
2. 10:01 - AI推荐维修方案: "更换空调压缩机"
3. 10:01 - Function "配件库存状态" 检查库存
   - 空调压缩机库存: 0个 (关键配件)
   - 返回: "缺货"
4. 10:01 - Automation "配件缺货自动通知" 触发
   - 发送通知到配件管理员: "【缺货】工单XX需要空调压缩机,当前库存为0"
   - 工单标记为"等待配件"
   - 创建配件补货提醒
5. 10:30 - 配件管理员采购配件,预计明天到货
6. 10:35 - 调度员通知客户: "您的车辆需要更换的配件暂时缺货,预计明天到货后立即为您维修"

**最终结果**: 提前发现配件问题,及时沟通客户,避免客户白跑一趟

### 场景3: 返修工单自动识别

**情况**: 同一辆车30天内第二次报修"发动机异响"

**系统响应**:
1. 客服创建新工单,VIN码相同,故障描述"发动机异响"
2. Function "是否一次修好" 自动检查
   - 查询历史工单: 发现30天前有工单,故障"发动机抖动,有异响"
   - 关键词匹配: "发动机" 重合
   - 返回: "返修"
3. Automation "疑似返修自动提醒" 触发
   - 发送告警到质检主管: "【返修告警】车辆XX疑似返修,原工单YY,请关注"
   - 发送通知到原技师: "您维修的车辆XX疑似返修,请关注"
   - 工单自动关联原工单
4. 质检主管查看原工单维修记录,发现可能是配件质量问题
5. 技师登记"返修",关联原工单,触发免费维修流程

**最终结果**: 及时发现返修问题,快速定位原因,提升客户满意度
