# 对象关联关系指南（Links Guide）

对象之间的关联关系（Link）是本体的"骨架"——它把孤立的业务对象连接成一张有意义的网络。

## 核心概念

### 什么是Link？
Link描述了两个业务对象之间的关系，类似于"谁和谁有什么关系"：
- 设备 **属于** 车间
- 工单 **关联** 设备
- 技师 **负责** 工单
- 供应商 **供应** 产品

### 为什么Link很重要？
- **导航能力**：从一台设备，快速找到它的所有工单、负责技师、配件清单
- **业务完整性**：删除设备时，相关工单怎么办？Link帮你管理这种级联关系
- **数据分析**：哪个供应商的配件故障率最高？需要跨对象的Link才能回答

## 三种关系类型

### 1. 一对一（One-to-One）
一个对象只关联一个目标对象。

**示例**：
```
员工 ←→ 工位
每个员工有一个固定工位，每个工位分配给一个员工
```

**特点**：较少见，通常意味着两个对象可以合并为一个

---

### 2. 一对多（One-to-Many）
一个对象关联多个目标对象，最常见的关系类型。

**示例**：
```
车间 ──→ 设备（多台）
一个车间有多台设备，每台设备只属于一个车间

设备 ──→ 工单（多个）
一台设备可以产生多个维修工单，每个工单只针对一台设备

供应商 ──→ 产品（多个）
一个供应商供应多种产品，每种产品来自一个主供应商
```

**特点**：最常见，几乎所有业务场景都会用到

---

### 3. 多对多（Many-to-Many）
两个对象之间互相有多个关联。

**示例**：
```
技师 ←→ 技能证书
一个技师可以持有多个证书，一种证书可以被多个技师持有

产品 ←→ 原材料
一种产品用到多种原材料，一种原材料被多种产品使用
```

**特点**：需要额外考虑关联的附加信息（如技师获得证书的日期）

---

## 关联关系设计要素

### 1. 关系方向
虽然关系是双向的，但需要明确"主方"：
```
✅ 设备 → 产生 → 工单（从设备角度看它的工单列表）
✅ 工单 → 针对 → 设备（从工单角度看它服务的设备）
两个方向都成立，选择更常用的那个作为主方向
```

### 2. 级联行为
当主对象被删除时，关联对象怎么办？

| 策略 | 行为 | 适用场景 |
|------|------|---------|
| 级联删除 | 主对象删除时，关联对象一起删除 | 订单删除→订单明细一起删除 |
| 置空引用 | 主对象删除时，关联字段设为空 | 员工离职→工单负责人置空 |
| 阻止删除 | 有关联对象时不允许删除主对象 | 供应商有进行中的订单→不能删除 |

### 3. 必要性
关联关系是否必须存在？

```
✅ 必要关联：工单必须关联一台设备（没有设备的工单没有意义）
✅ 可选关联：设备可以关联负责人（新设备可能暂未分配）
```

## 设计技巧

### 技巧1：从业务场景自然发现关系
不要硬想有哪些关系，而是从用户描述中自然提取：
- 用户说"技师去修设备" → 技师←→设备 有关系
- 用户说"这个订单是客户下的" → 订单→客户 有关系
- 用户说"一个车间有很多台设备" → 车间→设备 一对多

### 技巧2：先画全景图，再定细节
```
第一步：列出所有对象名称
第二步：用箭头连接有关系的对象
第三步：标注每条关系的类型（一对一/一对多/多对多）
第四步：标注关系的业务含义（属于/产生/负责/使用...）
```

### 技巧3：避免过度关联
不是所有对象都需要直接关联：
- ✅ 直接关联：设备→工单（强业务关系）
- ❌ 过度关联：设备→财务报表（可以通过工单间接关联）

### 技巧4：关系命名要有业务含义
```
✅ 好的命名："产生的工单"、"负责的技师"、"所属车间"
❌ 不好的命名："关联1"、"link_a"、"关系"
```

## 常见问题

### Q1: 什么时候用Link，什么时候用Property？
**判断标准**：
- 如果关联目标是另一个**独立管理的对象** → 用Link
  - 例：设备关联的"车间"（车间有自己的属性和操作）
- 如果只是记录一个**简单的值** → 用Property
  - 例：设备的"负责人姓名"（只是一个文本，不是独立的人员对象）

### Q2: 一个对象应该有多少个Link？
通常2-5个关联关系就够了。如果超过7个，考虑是否有些关系可以通过中间对象间接关联。

### Q3: 多对多关系需要中间表吗？
在本体配置中，多对多关系直接在Link中配置，不需要手动创建中间表。但如果关系本身有附加属性（如"技师的技能等级"），考虑创建中间对象。

### Q4: 关联的对象还没建模怎么办？
完全没问题。先记录关联关系，标注目标对象"待建模"。当前核心对象建完后，后续迭代再为关联对象建模。

## Link与其他要素的协同

```
Link在本体中的角色：

1. Property阶段识别Link：
   "这个字段引用了另一个对象" → 不是Property，是Link

2. Action中操作Link：
   "创建工单时自动关联当前设备" → Action需要设置Link
   "删除设备时解除所有工单关联" → Action需要处理级联

3. Automation中跨对象触发：
   "设备温度超标 → 自动为关联的所有技师发送通知"
   需要通过Link找到关联的技师
```
