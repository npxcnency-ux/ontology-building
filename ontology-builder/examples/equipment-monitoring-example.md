# 生产设备监控本体 - 完整示例

这是一个真实的业务场景示例,展示如何为生产车间的设备监控系统构建本体。

---

## 业务背景

**场景**: 某制造企业的生产车间有多台离心机、搅拌机等设备,需要实时监控设备状态,防止高温故障,并管理设备保养。

**痛点**:
- 设备温度异常时,需要人工巡检才能发现
- 设备报废流程复杂,需要多部门审批
- 保养计划经常被遗忘,导致设备寿命缩短

**目标**:
- 温度异常自动报警并停机
- 报废审批流程自动化
- 自动提醒即将到期的保养计划

---

## 第一步:定义业务对象

**对象名称**: 生产设备 (Production Equipment)

**关键角色**:
- 运维人员:监控设备状态,处理故障
- 设备经理:审批报废申请,管理保养计划
- 技术员:执行设备维护

## 第三步:计算规则 (Function)

### Function 1: 设备温度风险等级
**业务目的**: 根据当前温度自动判断设备风险

**计算逻辑**:
```
如果 当前温度 为空 或 温度标准 为空：
  返回 "未知"（数据缺失，无法判断）
如果 当前温度 > 温度标准：
  返回 "高风险"
否则如果 当前温度 > 温度标准 × 0.8：
  返回 "中风险"
否则：
  返回 "低风险"
```

**示例**:
- 当前温度 75°C, 标准 90°C → 低风险
- 当前温度 87°C, 标准 90°C → 中风险 (87 > 90×0.8=72)
- 当前温度 95°C, 标准 90°C → 高风险
- 当前温度 为空 → 未知

---

### Function 2: 设备是否在线
**业务目的**: 判断设备通信是否正常

**计算逻辑**:
```
如果 最后心跳时间 为空：
  返回 "未知"（从未收到心跳）
时间差 = 当前时间 - 最后心跳时间（分钟）
如果 时间差 < 5分钟：
  返回 "在线"
否则：
  返回 "离线（XX分钟）"
```

**示例**:
- 最后心跳: 2024-03-18 14:28, 当前: 14:30 → 在线
- 最后心跳: 2024-03-18 14:20, 当前: 14:30 → 离线 (10分钟)

---

### Function 3: 是否需要保养
**业务目的**: 根据运行时长提醒保养

**计算逻辑**:
```
距上次保养天数 = 当前日期 - 上次保养日期

如果 累计运行时长 >= 1000小时 或 距上次保养天数 >= 90天：
  返回 "需要保养"
否则如果 累计运行时长 >= 800小时 或 距上次保养天数 >= 75天：
  返回 "即将到期"
否则：
  返回 "正常"
```

## 第四步:操作按钮 (Action)

### Action 1: 创建新设备
**操作类型**: 创建

**必填字段**:
- 设备编号 (String, 必须唯一)
- 设备名称 (String)
- 设备类型 (Enum: 离心机/搅拌机/压缩机)
- 所属车间 (String)
- 安装日期 (Date)
- 温度标准 (Double)

**权限**: 设备管理员

**执行效果**:
1. 创建设备对象
2. 初始状态 = "待启用"
3. 创建时间 = 当前时间
4. 创建人 = 当前用户

---

### Action 2: 变更设备状态
**操作类型**: 更新

**可修改字段**:
- 设备状态 (Enum: 运行中/停机/维修中/已报废)
- 变更原因 (String, 必填)

**前置条件**:
- 当前状态不能是"已报废"

**权限**:
- 运维人员: 可变更为"停机"或"维修中"
- 设备经理: 可变更为任何状态

**执行效果**:
1. 更新设备状态
2. 记录变更时间和变更人
3. 追加到状态变更日志

---

### Action 3: 紧急停机
**操作类型**: 业务操作

**必填字段**:
- 停机原因 (Enum: 设备故障/安全隐患/计划维护/其他)

**前置条件**:
- 设备状态必须是"运行中"

**权限**:
- 高级技术员及以上
- 仅限工作时间 (8:00-20:00)

**二次确认**: "确定紧急停机?此操作将立即断电。"

**执行效果**:
1. 发送停机指令到设备控制系统
2. 设备状态 → "停机"
3. 记录停机时间
4. 如果原因是"设备故障",自动创建故障报告工单
5. 发送企业微信通知: "【紧急】XX设备已停机"

---

### Action 4: 申请设备报废
**操作类型**: 业务操作

**必填字段**:
- 报废原因 (Enum: 老化/损坏/淘汰/其他)
- 详细说明 (String)

**前置条件**:
- 设备状态必须是"停机"
- 设备使用年限 >= 5年 OR 累计故障次数 >= 10次

**权限**: 设备管理员

**执行效果**:
1. 创建报废审批工单
2. 指定审批人为设备经理
3. 发送审批通知 (企业微信 + 邮件)
4. 等待审批结果

---

### Action 5: 预约保养
**操作类型**: 业务操作

**必填字段**:
- 保养类型 (Enum: 日常保养/深度保养/年检)
- 预约日期 (Date, 必须 >= 当前日期 + 3天)
- 预约时段 (Enum: 上午/下午/晚上)

**前置条件**:
- 设备状态不能是"已报废"

**权限**: 设备负责人或运维人员

**执行效果**:
1. 检查该时段是否已有预约
2. 创建保养工单
3. 自动分配保养工程师
4. 发送提醒给工程师和设备负责人
5. 在日历上标记保养时段

---

## 第五步:自动化规则 (Automation)

### Automation 1: 高温自动保护机制
**类型**: 异常响应

**触发条件**:
- Function "设备温度风险等级" 返回 "高风险"

**自动流程**:
1. 调用 Action "紧急停机"
   - 停机原因 = "安全隐患"
2. 创建故障报告工单
   - 工单标题: "设备 XX 高温紧急停机"
   - 优先级: 紧急
3. 发送多渠道通知
   - 企业微信 → 设备负责人 + 运维经理
   - 短信 → 值班工程师
   - 邮件 → 设备经理 (带温度趋势图)
4. 记录事件日志

**限流配置**:
- 同一设备1小时内最多触发1次 (避免频繁停机)

**错误处理**:
- 策略: 自动重试（最多3次，间隔60秒）+ 通知管理员
- 如果停机指令发送失败 → 重试 + 短信通知值班工程师人工处理

---

### Automation 2: 设备离线告警
**类型**: 异常响应

**触发条件**:
- Function "设备是否在线" 返回 "离线"且离线时长 > 10分钟

**自动流程**:
1. 发送告警通知
   - 企业微信 → 运维值班群
   - 消息: "【告警】XX设备离线 XX 分钟,请检查通信"
2. 记录离线开始时间

**限流配置**:
- 离线期间每30分钟提醒一次

**错误处理**:
- 策略: 直接跳过（非关键提醒，下次触发时会再通知）

---

### Automation 3: 保养到期提醒
**类型**: 定时任务

**触发时间**: 每天早上 8:00

**自动流程**:
1. 查询所有设备,调用 Function "是否需要保养"
2. 筛选出返回"需要保养"或"即将到期"的设备
3. 生成保养提醒列表
4. 发送每日保养报告
   - 邮件 → 设备经理
   - 企业微信 → 运维团队
5. 对于"需要保养"的设备,自动创建保养提醒工单

**错误处理**:
- 策略: 通知管理员（邮件发送失败时，记录日志并在下次定时执行时重试）

---

### Automation 4: 报废审批流程编排
**类型**: 流程编排

**触发事件**:
- Action "申请设备报废" 被执行

**自动流程**:
1. 创建审批工单
   - 审批人: 设备经理
   - 附件: 报废申请表 + 设备历史故障记录
2. 发送审批通知
   - 企业微信 → 设备经理: "【待审批】XX 申请报废设备 YY"
3. 等待审批结果 (超过3天未审批,自动提醒)
4. 审批通过后:
   - 设备状态 → "已报废"
   - 创建资产处置工单
   - 通知财务部门更新资产账目
   - 发送完成通知给申请人
5. 审批拒绝后:
   - 设备状态恢复为"停机"
   - 通知申请人拒绝原因

**错误处理**:
- 策略: 自动重试（最多3次）+ 通知管理员
- 通知财务部门失败 → 重试 + 通知系统管理员人工处理

## 典型使用场景

### 场景1: 设备温度异常自动保护

**情况**: 2#离心机温度从87°C突然升至95°C

**系统响应**:
1. Property "设备温度" 更新为 95°C
2. Function "设备温度风险等级" 自动计算 → "高风险"
3. Automation "高温自动保护机制" 被触发
4. Action "紧急停机" 自动执行
   - 硬件立即断电
   - 设备状态 → "停机"
5. 通知发送:
   - 企业微信 → 张三 (设备负责人): "【紧急】2#离心机高温停机"
   - 短信 → 15912345678 (值班工程师)
   - 邮件 → 李经理 (带24小时温度趋势图)
6. 故障工单自动创建

**最终结果**: 设备在温度异常后2秒内自动停机,避免了更严重的损坏。工程师收到通知后5分钟到达现场处理。

---

### 场景2: 设备报废审批流程

**情况**: 1#离心机使用8年,故障频繁,运维人员申请报废

**系统响应**:
1. 运维人员点击 Action "申请设备报废"
   - 填写报废原因: "老化"
   - 填写详细说明: "设备已使用8年,近3个月故障5次,维修成本过高"
2. 系统检查前置条件:
   ✅ 设备状态 = "停机" (满足)
   ✅ 使用年限 8年 >= 5年 (满足)
3. Action执行成功
4. Automation "报废审批流程编排" 被触发:
   - 创建审批工单
   - 企业微信通知设备经理: "【待审批】张三申请报废1#离心机"
5. 设备经理收到通知,查看工单:
   - 审批工单显示: 报废原因、设备历史故障记录 (自动附加)
6. 设备经理点击"审批通过"
7. Automation继续执行:
   - 设备状态 → "已报废"
   - 创建资产处置工单
   - 通知财务部门
   - 通知张三: "报废申请已通过"

**最终结果**: 整个报废审批流程自动化,从申请到完成只需2小时,原本需要3-5天的流程大幅缩短。

## 配置要点总结

| 配置项 | 数量 | 关键点 |
|--------|------|--------|
| Property | 12个 | 3个时序属性 (温度、运行时长、心跳时间) |
